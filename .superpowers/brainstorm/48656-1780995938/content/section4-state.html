Section 4 — State Management

BLoC for complex async flows · Cubit for simple state · shared base classes for consistency

Base state pattern

core/bloc/app_state.dart
sealed class AppState<T> extends Equatable { const AppState(); } class InitialState<T> extends AppState<T> { const InitialState(); @override List get props => []; } class LoadingState<T> extends AppState<T> { const LoadingState(); @override List get props => []; } class SuccessState<T> extends AppState<T> { const SuccessState(this.data); final T data; @override List get props => [data]; } class ErrorState<T> extends AppState<T> { const ErrorState(this.failure); final Failure failure; @override List get props => [failure]; }

Auth BLoC (example)

auth_bloc.dart
class AuthBloc extends Bloc<AuthEvent, AuthState> { AuthBloc({ required LoginUseCase loginUseCase, required RegisterUseCase registerUseCase, required SubmitKycUseCase submitKycUseCase, }) : super(const AuthInitial()) { on<LoginRequested>(_onLoginRequested); on<RegisterRequested>(_onRegisterRequested); on<KycSubmitted>(_onKycSubmitted); } Future<void> _onLoginRequested( LoginRequested event, Emitter<AuthState> emit, ) async { emit(const AuthLoading()); final result = await _loginUseCase( LoginParams(email: event.email, password: event.password), ); result.fold( (failure) => emit(AuthError(failure)), (user) => emit(AuthSuccess(user)), ); } }

BLoC vs Cubit decision

Feature Type Why
authBLoCMultiple events, KYC steps, side-effects
homeCubitLoad dashboard data, no complex events
product_managementBLoCCRUD events, pagination, image upload
transactionsBLoCFilters, pagination, date range events
refer_and_earnCubitLoad referral code + stats, share action
invoicesBLoCList + filter + download events

AppBlocObserver

app/app_bloc_observer.dart
class AppBlocObserver extends BlocObserver { @override void onError(BlocBase bloc, Object error, StackTrace stackTrace) { logger.e(error, stackTrace: stackTrace); // analytics.logError() in prod super.onError(bloc, error, stackTrace); } @override void onChange(BlocBase bloc, Change change) { if (FlavorConfig.instance.enableLogging) logger.d('${bloc.runtimeType}: $change'); super.onChange(bloc, change); } }
Global error catch: AppBlocObserver is set in bootstrap() via Bloc.observer = AppBlocObserver(). All unhandled BLoC errors flow here — logged in dev, sent to analytics in prod.
